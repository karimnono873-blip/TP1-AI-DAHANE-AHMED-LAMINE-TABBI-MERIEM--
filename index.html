1. Problématique & Notions Théoriques

Une entreprise souhaite classer automatiquement ses clients pour adapter sa stratégie de marketing. Le jeu de données utilisé est le Mall Customer Dataset. L'objectif est de segmenter les clients selon leurs revenus en utilisant un algorithme d'apprentissage supervisé : Random Forest (Forêt Aléatoire).

Qu'est-ce que le Random Forest ?

Le Random Forest est un algorithme de type Ensemble Learning (apprentissage ensembliste). Son principe est basé sur la technique du Bagging (Bootstrap Aggregating) :

  • Création d'arbres multiples : L'algorithme crée une "forêt" de plusieurs arbres de décision indépendants.
  • Bootstrap : Chaque arbre est entraîné sur un sous-ensemble aléatoire des données (avec remise).
  • Sélection de features : À chaque nœud d'un arbre, seul un sous-ensemble aléatoire des variables (features) est testé pour diviser les données.
  • Agrégation (Vote) : Pour la classification, la prédiction finale est déterminée par un vote majoritaire de tous les arbres. Cela réduit considérablement la variance et évite le surapprentissage (overfitting).

2. Préparation des Données et Évaluation

Création des Labels et Nettoyage

Le jeu de données original n'est pas labellisé pour la classification. Nous devons créer notre propre cible (Target). Nous allons segmenter la colonne Annual Income (k$) en 3 classes : Faible, Moyen, et Élevé.

Pourquoi supprimer l'ID et la cible des Features ?

Avant d'entraîner le modèle, nous devons isoler les variables explicatives (Features : X) de la variable à prédire (Target : y).
Il est impératif de supprimer la colonne Cible de X (sinon le modèle triche en regardant la réponse). Il faut également supprimer la colonne CustomerID. L'ID est un identifiant unique aléatoire qui ne porte aucune information comportementale. S'il est laissé, le modèle pourrait essayer d'y trouver une fausse corrélation (bruit), perturbant ainsi l'apprentissage.

Les Métriques d'Évaluation

  • Matrice de Confusion : Un tableau qui résume les performances du modèle en croisant les valeurs réelles et les prédictions (Vrais Positifs, Faux Positifs, Vrais Négatifs, Faux Négatifs).
  • Précision (Precision) : Parmi toutes les prédictions positives faites par le modèle, combien étaient réellement correctes ?
  • Rapport de Classification : Un résumé textuel incluant la précision, le rappel (recall) et le F1-score pour chaque classe.

3. Implémentation Python (Solution Complète)

Voici le script permettant de résoudre ce TP, du chargement des données jusqu'à l'évaluation du modèle :


import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report, confusion_matrix
from sklearn.preprocessing import LabelEncoder

# 1. Chargement des données
df = pd.read_csv('Mall_Customers.csv')

# 2. Labellisation (Création des 3 classes de revenus)
# Catégorisation : < 40k (Faible), 40k-70k (Moyen), > 70k (Élevé)
bins = [0, 40, 70, 150]
labels_revenu = ['Faible', 'Moyen', 'Élevé']
df['Income_Class'] = pd.cut(df['Annual Income (k$)'], bins=bins, labels=labels_revenu)

# 3. Encodage des variables catégorielles (Gender)
le = LabelEncoder()
df['Gender'] = le.fit_transform(df['Gender']) # Male=1, Female=0

# 4. Séparation Features (X) et Target (y)
# On supprime CustomerID (inutile) et les colonnes de revenus (pour éviter la triche)
X = df.drop(['CustomerID', 'Annual Income (k$)', 'Income_Class'], axis=1)
y = df['Income_Class']

# 5. Split Train/Test (70% apprentissage, 30% test)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 6. Initialisation et Entraînement du modèle Random Forest
rf_model = RandomForestClassifier(n_estimators=100, random_state=42)
rf_model.fit(X_train, y_train)

# 7. Prédictions sur l'ensemble de test
y_pred = rf_model.predict(X_test)

# 8. Évaluation du modèle
print("=== Matrice de Confusion ===")
print(confusion_matrix(y_test, y_pred))

print("\n=== Rapport de Classification ===")
print(classification_report(y_test, y_pred))
